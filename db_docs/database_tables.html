WorkBee Application Database Schema

To copy these to MS Word: Select All (Ctrl+A), Copy (Ctrl+C), and Paste (Ctrl+V) into your document. The tables will preserve their structure perfectly.

1. Table: auth_user (Django Built-in User)

Field Name Data Type Constraints / Description
id Integer Primary Key, Auto-increment
username Varchar(150) Unique, Required. Identifier for login.
password Varchar(128) Required. Hashed password string.
email Varchar(254) Optional/Required depending on config.
is_active Boolean Default: True. Designates if account is active.
date_joined DateTime Automatically set on creation.

2. Table: profiles

Field Name Data Type Constraints / Description
id Integer Primary Key, Auto-increment
user_id Integer Foreign Key (auth_user.id), Unique (One-to-One), CASCADE delete
role Varchar(10) Choices: 'seeker', 'provider'
bio LongText Optional description of the user/company.
location Varchar(100) Optional physical location.
created_at DateTime Auto-generated timestamp.

3. Table: jobs

Field Name Data Type Constraints / Description
id Integer Primary Key, Auto-increment
title Varchar(200) Required. Name of the job position.
description LongText Required. Full details of the job.
location Varchar(100) Optional. E.g. 'Remote', 'New York'.
created_by_id Integer Foreign Key (auth_user.id), CASCADE delete. The provider.
is_active Boolean Default: True. Whether job is open.
created_at DateTime Auto-generated on creation.
updated_at DateTime Auto-updates on modification.

4. Table: saved_jobs

Field Name Data Type Constraints / Description
id Integer Primary Key, Auto-increment
job_id Integer Foreign Key (jobs.id), CASCADE delete.
user_id Integer Foreign Key (auth_user.id), CASCADE delete. The seeker.
saved_at DateTime Auto-generated timestamp.
Note: Unique constraint on the combination of (job_id, user_id).

5. Table: applications

Field Name Data Type Constraints / Description
id Integer Primary Key, Auto-increment
job_id Integer Foreign Key (jobs.id), CASCADE delete.
applicant_id Integer Foreign Key (auth_user.id), CASCADE delete. The seeker.
cover_letter LongText Optional. Message to the employer.
status Varchar(10) Choices: 'pending', 'accepted', 'rejected'. Default: 'pending'.
applied_at DateTime Auto-generated timestamp.
Note: Unique constraint on the combination of (job_id, applicant_id) to prevent duplicate applications.
